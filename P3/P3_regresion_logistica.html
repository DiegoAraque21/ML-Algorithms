1. Clasificador Regresin Logstica

El clasificador Regresin Logstica se ha implementado en el fichero ClasificadorRegresionLogistica.py, siendo una clase que hereda de Clasificador y que implementa los mtodos de entrenamiento y clasifica.

Este clasificador lineal tiene como objetivo conseguir el hiperplano que mejor separe las clases maximizando la verosimilitud de los datos. Esto se consigue mediante el algoritmo del descenso por gradiente en el cual se van actualizando los puntos del hiperplano en cada iteracin hasta que se alcanza un mnimo.

La probabilidad de que un dato pertenezca a una clase u otra se calcula mediante la funcin sigmoide, la cual se define como:

$$\sigma(z) = \frac{1}{1+e^{-z}}$$

donde $z$ es la funcin lineal que se define como:

$$z = w_0 + w_1x_1 + w_2x_2 + ... + w_nx_n$$

donde $w_0$ es el trmino independiente y $w_1, w_2, ..., w_n$ son los pesos de cada caracterstica.

Cada vez que se termina de aplicar el algoritmo del descenso en el dataset, se completa una poca. Cuando se terminan un determinado nmero de pocas, se llega a unos valores determinados de los pesos que en teora seran los ptimos para la clasificacin.

En nuestro clasificador, el algoritmo del descenso por gradiente se aplica en el mtodo entrenamiento para conseguir los pesos finales que se validarn con los datos de test en el mtodo clasifica. Ya que se usa la funcin sigmoide para calcular la probabilidad, el umbral para clasificar una dato en una clase u otra es 0.5.

Adems, tambin se aplica la funcionalidad de preproceso de los datos mediante la normalizacin de stos que ya hemos observado en prcticas anteriores que supone una mejora en el rendimiento de los clasificadores.

En este caso, hemos optado por probar el clasificador Regresion Logistica con la estrategia de particionado de Validacin Simple ya que con las anteriores prcticas apenas hemos observado diferencias entre los resultados obtenidos con esta estrategia y la de Validacin Cruzada.

# hold-out validation (10 samples)

vs = ValidacionSimple(10, 0.3)

learning_rates = [0.01, 0.02, 0.1, 0.25, 0.5, 0.75, 1]
epochs = [50, 100, 1000]

combinations = {
    0.01: {},
    0.02: {},
    0.1: {},
    0.25: {},
    0.5: {},
    0.75: {},
    1: {}
}

for lr in learning_rates:
    for epoch in epochs:

        # create classifiers
        mylr = ClasificadorRegresionLogistica(l_rate=lr, n_epochs=epoch, normalize=True)
        mylr_not = ClasificadorRegresionLogistica(l_rate=lr, n_epochs=epoch)

        # validate classifiers
        mylr_heart = mylr.validacion(vs, heart, mylr)
        mylr_heart_not = mylr_not.validacion(vs, heart, mylr_not)

        # get accuracy and error rate
        accuracy_vs_heart = np.mean(1 - np.array(mylr_heart))
        accuracy_vs_heart_not = np.mean(1 - np.array(mylr_heart_not))

        error_rate_vs_heart = 1 - accuracy_vs_heart
        error_rate_vs_heart_not = 1 - accuracy_vs_heart_not

        # save results
        combinations[lr][epoch] = {
            'accuracy_vs_heart': accuracy_vs_heart,
            'accuracy_vs_heart_not': accuracy_vs_heart_not,
            'error_rate_vs_heart': error_rate_vs_heart,
            'error_rate_vs_heart_not': error_rate_vs_heart_not,
        }

Como ya hemos comentado en otras prcticas, estos resultados son muy esperables ya que los tipos de atributos influyen mucho en el cmputo de las probabilidades y en la clasificacin de los datos.

Por otro lado, respecto a la clasificacin y a los hiperparmetros a modificar, se puede resaltar como ptima configuracin la combinacin de l_rate = 0.02 y n_epochs = 1000, ya que es la que obtiene un porcentaje de acierto ms alto. Adems de esta, hay muchas ms opciones que producen altos procentajes de acierto como: l_rate = 0.02 y epoch = 50 o l_rate = 1 y epoch = 1000.

Un nmero pequeo de tasa de aprendizaje supone que el algoritmo del descenso por gradiente dar saltos ms pequeos cada vez que se actualicen los datos, mientras que un nmero muy alto implica mayor velocidad a la hora de minimizar. Esta combinacin de hiperparmetros influye en la velocidad a la que se llega a la convergencia y en el nmero de iteraciones que se realizan.

Espacio ROC

Para evaluar la precisin de los clasificadores, usaremos el espacio ROC. Este espacio nos permite comparar la precisin de los clasificadores. Por ello, el paso principal es el clculo de la matriz de confusin. Esta matriz se basa en el clculo de los verdaderos positivos, verdaderos negativos, falsos positivos y falsos negativos. Una vez obtenidos estos valores, calculamos la tasa de falsos positivos (valor en eje X) y la tasa de verdaderos positivos (valor en eje Y). Con estos valores ya podemos dibujar es espacio ROC. Cuanto ms cerca estn los puntos al (0,1), mejor precisin tendr le clasificador.

A continuacin, representamos un espacio ROC para cada conjunto de datos, con dos puntos por clasificador segn su estrategia de particionado.

4. Curva ROC para Regresin Logstica

Como ya sabemos, el clasificador de Regresin Logstica es un clasificador lineal que se basa en la funcin sigmoide para calcular la probabilidad de que un dato pertenezca a una clase u otra. Esta funcin sigmoidal, especifica un umbral para tomar una decisin u otra segn un dato de test o no.

Para poder encontrar el umbral ptimo, se utiliza la curva ROC. La manera ms rudimentaria de encontrar ese valor, sera ir variando el umbral hasta que se obtenga el mejor resultado, pero supone un nivel de computacin muy alto. Por ello, nuestra implementacin se basa en el algoritmo propuesto por Fawcett, en el cual se compara el score obtenido por cada dato de test con el umbral y si este ha variado, se actualiza el umbral.

Hemos elegido el conjunto de datos de WDBC, ya que todos los atributos son numricos y supone una gran ventaja a la hora de usar los clasificadores. Como hemos visto antes, la tasa de acierto para este dataset rondaba el 90%.
